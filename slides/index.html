Pre-Reqs

In order to create our first site, we need to determine where to place it and install a few pre-reqs.

  • Install both Apache2 and Git
  • Our site will be created under /var/www/html

Our First Site!

root@ubuntu-xenial:/var/www/html# hugo new site mysite

Congratulations! Your new Hugo site is created in /var/www/html/mysite.

Just a few more steps and you're ready to go:

  1. Download a theme into the same-named folder. Choose a theme from https://themes.gohugo.io/, or create your own with the "hugo new theme <THEMENAME>" command.
  2. Perhaps you want to add some content. You can add single files with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
  3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

Installing a Theme

Installing a theme can be completed in three steps. We'll dig further into themes in a later section.

cd /var/www/html/mysite
git init
git submodule add \
https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
echo 'theme = "ananke"' >> config.toml

Create a First Post

We can create a new post through the hugo cli command. Afterwards, edit the file to include content.

# hugo new posts/my-first-post.md
/var/www/html/mysite/content/posts/my-first-post.md created

Start the Built In Server

Hugo has a built-in server that runs on port 1313. We add the "-D" flag so we build documents in a draft status.

# hugo server -D

It is possible to configure which interface Hugo will bind to when running its server

# hugo server --bind $(hostname -I) -D

Closer Look - Server Output

The Directory Structure

# ls -1
archetypes
config.toml
content
data
layouts

archetypes

Content file types are stored in this directory. Viewing the default.md files gives us an example of an archetype in action.

config.toml / config

The default configuration is config.toml, but as we get more in-depth with our Hugo use, there may come at time where might need a stating and production configuration. Creating multiple files in a config directory is how we would do this.

content

This is where the content for your website will live.

data

The data directory stores configuration files. For our examples these files will be essentialy datasets which will be used to generate lists in our HTML.

layouts

Stores HTML templates to outline how content will be rendered.

Some types of layouts

  • list pages
  • the homepage
  • partials
  • single page templates

static

A place for all your static content e.g. stylesheets, JavaScript, images etc.

themes

The default directory to store themes.

The wordcount function takes input and counts the words.

Example - cond

We can return text based on a condition being true or false

{{ cond (eq (len $geese) 1) "goose" "geese" }}
